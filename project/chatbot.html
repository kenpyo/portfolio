チャットボット開発

「幹事の負担を最小限にする」ことを目標にLINEのグループチャット上で動作する幹事代行チャットボットを開発しました。
開発は要件定義から詳細設計、実装までを4人のチームで行いました。

使用技術

  • AI / Model: Gemini API
  • Library / API: LINE Messaging API, Hot Pepper API
  • Logic / Tools: Python

開発のプロセスと工夫

アジャイル開発でコア機能から順次追加、改善を繰り返しました。

1. Gemini API による自然な対話の実装

定型文以外の入力に対してもGemini APIを介して解析を行い、ユーザーが利用したい機能の推測、またその機能に必要な情報の抜き出しを行い自然な会話の中で複数の機能を利用可能にしました。
普段の会話を邪魔しないために@on,@offのコマンドにより機能の稼働、停止を操作します

2. 複数APIを連携させた店舗検索ロジック

入力をGeminiでJSON形式に構造化し、必要なパラメータを抽出。これをHot Pepper APIへ受け渡し店舗検索を行う機能を実装しました。
「近くの居酒屋」や「新宿で個室の焼肉を探して」と送られると条件に対応する店舗の検索結果上位5つを表示します。
工夫点として、「博多でラーメン」と送られた際、[博多][ラーメン]の二つの情報を抜き出すので博多以外の地域の博多ラーメンが出てしまっていましたが、地域の情報を必須とすることで解決しました。 また、「近くの」などの文言があった場合に位置情報送信用のボタンを表示し位置情報を利用して検索できるようになっています。

3. Googleカレンダー同期による予定管理

Google Calendar APIと連携し、グループメンバーの予定を取得し空き日を表示、グループチャット上からカレンダーに予定を追加が可能です。
「いつが空いてる？」「〇日の〇時に集合」などで、LINEを起点としたシームレスな予定管理を可能にしました。

4. 投票機能の実装

検索した店舗の中から投票したり、予定の候補日から投票することが可能です
